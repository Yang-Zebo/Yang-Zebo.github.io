Class-类 ​

TypeScript完全支持ES2015中引入的class关键字，TypeScript添加了类型注释和其他语法，以允许你表达类和其他类型之间的关系。

1. 类成员 ​

类型注释是可选的，可以写在属性声明后面，但如果未指定，则为隐式any。

typescript
   // 创建公共可写属性
class Point {
  x: number
  y: number
}

const pt = new Point()
pt.x = 0
pt.y = '0' // error

可以在tsconfig.json配置strictPropertyInitialization ，启用此选项时编译器会要求类的属性在构造函数中被明确初始化。或者显示地声明属性为undefined类型或者使用非空断言。

typescript
   // 开启 strictPropertyInitialization 之后, 构造函数中初始化，或者其他写法
class Point1 {
  name: string

  constructor(name: string = '') { // 在构造函数中始终提供一个默认值
    this.name = name
  }
}

class Point2 {
  name: string | undefined // 显式允许 undefined
  constructor(name?: string) {
    this.name = name
  }
}

class Point3 {
  name!: string // 使用非空断言操作符，它会绕过类型检查
  constructor(name?: string) {
    this.name = name
  }
}

字段可以以readonly修饰符作为前缀，这可以防止对构造函数之外的字段进行赋值。

typescript
   class Greeter {
  readonly name: string = 'world'

      this.name = otherName // ok
    }

const g = new Greeter()
g.name = 'also not ok' // error，只读属性只能在构造函数中赋值

类构造函数的类型注释与函数非常相似，可以添加带有类型注释、默认值和重载的参数。

   class Point1<T> {

    this.x = x
    this.y = y
  }

class Point2 {
  x: number = 0
  y: number = 0

  constructor(x: number, y: number)
  constructor(x: string)
  constructor(x: string | number, y: number = 0) {
  }

2. 类继承 ​

implements ​

implements从句来检查一个类是否满足特定的interface如果一个类未能正确实现它，则会触发错误。

  • 类也可以实现多个接口，class C implements A, B {}。
  • implements子句只是检查类可以被视为接口类型，它不会改变类的类型或其方法。
  • 使用可选属性实现接口同样不会创建该属性。
typescript
interface Pingable {
  ping(): void;
}

class Sonar implements Pingable {
  ping() {
    console.log('ping!')
  }
}

/* error 未实现接口
class Ball implements Pingable { 
  pong() {
    console.log('ping!')
  }
}*/

extends ​

extends从句来继承一个类。派生类具有其基类的所有属性和方法，还可以定义额外的成员。

typescript
class Animal {
  move() {
    console.log('Moving along!')
  }
}

class Dog extends Animal {
  woof() {
    console.log('woof!')
  }
}

const d = new Dog()
d.move() // Moving along
d.woof() // woof!

useDefineForClassFields是TypeScript编译器的一个选项，它影响了类字段的初始化方式。

  • ES2022及更高版本时，类字段可以直接在类中声明并初始化，而无需在构造函数中显式赋值。
  • 当这个选项设置为true，TypeScript会使用Object.defineProperty()来定义类字段，而不是简单地赋值。
  • 当这个选项设置为true，使用extent继承覆盖基类字段可能会出问题，可以写declare来向表明这个字段不应该有运行时影响。

3. 成员可见性 ​

public ​

类成员的默认可见性为public，可以在任何地方访问。

  • public已经是默认的可见性修饰符，所以不需要在类成员上编写它。
typescript
class Animal {
  public name: string

  constructor(name: string) {
    this.name = name
  }
}

protected ​

当类成员被标记为protected时，它可以在声明它的类内部以及任何继承自该类的子类中被访问。但是，它不能从类的外部被访问。

typescript
class ParentClass {
  protected myProtectedField: string = 'I am protected'

  public showProtectedField() {
    console.log(this.myProtectedField) // 合法，因为这是在类内部访问  
  }
}

class ChildClass extends ParentClass {
  public accessProtectedField() {
    console.log(this.myProtectedField) // 合法，因为这是在子类中访问  
  }
}

const obj = new ParentClass()
console.log(obj.myProtectedField) // 错误，不能从类外部访问  

const childObj = new ChildClass()
console.log(childObj.myProtectedField) // 错误，尽管是子类实例，但仍然不能从外部直接访问  
childObj.accessProtectedField() // 合法，通过子类的方法访问

private ​

当类成员被标记为private时，它只能在声明它的类内部被访问。

  • private 成员对派生类不可见，所以派生类不能增加它们的可见性。
  • private 允许在类型检查期间使用括号表示法进行访问。
typescript
class Person {
  private name: string = 'John'
}

class MyClass extends Person {
  private myPrivateField: string = 'I am private'

  public showPrivateField() {
    console.log(this.name) // 错误，不能从派生类中访问
    console.log(this.myPrivateField) // 合法，因为这是在类内部访问  
  }
}

const obj = new MyClass()
console.log(obj.myPrivateField) // 错误，不能从类外部访问
console.log(obj['myPrivateField']) // OK
console.log(obj.name) // 错误，不能从派生类中访问

4. 静态成员 ​

  • 静态成员是类本身而不是类的实例上的成员。它们通过static关键字进行定义。
  • 静态成员也可以使用相同的public、protected和private可见性修饰符。
  • 静态成员也能被继承。
typescript
class Base {
  static age = 10
}

class MyClass extends Base {
  private static x = 0

  static printX() {
    console.log(MyClass.x)
  }
}

console.log(MyClass.x) // error
MyClass.printX() // OK
console.log(MyClass.age) // OK

5. 泛型类 ​

  1. 类可以像接口一样使用泛型约束和默认值。
  2. 泛型类的static成员永远不能引用类的类型参数。
typescript
class Box<Type> {

  constructor(value: Type) {

const b = new Box('hello!')

6. 类中的this ​

this参数 ​

TypeScript检查是否使用正确的上下文调用带有this参数的函数。

typescript
class MyClass {
  name = 'MyClass'
  age = 10
  getAge: (this: MyClass) => void = () => {
    return this.age
  }

  getName(this: MyClass) {
    return this.name
  }
}

const c = new MyClass()
let obj = {
  getName: c.getName,
  getAge: c.getAge
}
c.getName() // OK
obj.getName() // error this 指向对象 obj 而不是约束的 MyClass

this类型 ​

在TypeScript中，当定义一个类并在其方法中使用this关键字时，this的类型自动被推断为当前类的实例类型。

typescript
class Person {
  name: string
  age: number

  constructor(name: string, age: number) {
    this.name = name
    this.age = age
  }

  greet() {
    console.log(`Hello, my name is ${ this.name } and I am ${ this.age } years old.`)
  }

  // 假设我们想在某个方法中引用当前类的类型  
  describeMyself(): Person {
    // 这里 `this` 的类型就是 Person  
    console.log(`Describing myself: ${ this.name }, ${ this.age }`)
    return this // 这里返回的是 Person 类型的实例  
  }
}

const person1 = new Person('Alice', 30)
person1.greet() // 输出: Hello, my name is Alice and I am 30 years old.  
const selfDescription = person1.describeMyself() // selfDescription 的类型是 Person  
console.log(selfDescription instanceof Person) // 输出: true

8. 构造函数签名 ​

InstanceType工具类型是一个非常有用的泛型工具，能够从构造函数类型(也称为类类型)中获取一个类的实例类型。

  • 当想要表示一个类实例化后得到的对象的类型时特别有用。
typescript
class Person {
  name: string
  age: number

  constructor(name: string, age: number) {
    this.name = name
    this.age = age
  }
}

// 想要表达一个变量是 Person 类实例的类型，我们可以直接写
let p1: Person = new Person('Alice', 30)
// 想要表达一个更通用的概念，比如“某个未知类的实例类型”，那么InstanceType就派上用场了
// typeof Person 获取了 Person 类的类型(注意，这是类的类型，而不是其实例的类型)
// 然后 InstanceType<typeof Person> 将这个类的类型转换为其实例的类型
// PersonInstance 类型实际上就是 Person 的实例类型，与直接写 Person 作为类型是一样的
type PersonInstance = InstanceType<typeof Person>
let p2: PersonInstance = new Person('Alice', 30)

9. 抽象类 ​

  1. TypeScript中，abstract关键字用于定义一个抽象类或抽象类中的抽象方法。抽象类是一种不能被实例化的类，它通常用作基类也称为父类或超类，用于被其他子类继承。
  2. 抽象方法是声明在抽象类中的方法，但它没有实现体，即没有方法体的大括号{}。
typescript
// 定义一个抽象类  
abstract class Animal {
  // 抽象方法  
  abstract makeSound(): void;

  // 非抽象方法（可选）  
  move(): void {
    console.log('The animal moves.')
  }
}

// 继承抽象类并实现抽象方法  
class Dog extends Animal {
  makeSound(): void {
    console.log('Woof!')
  }
}

let myDog = new Dog()
myDog.makeSound() // 输出: Woof!  
myDog.move() // 输出: The animal moves.
